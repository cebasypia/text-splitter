# 003: トースト通知システムの導入

## 概要
ユーザーがチャンクをコピーした際などのアクションに対して、視覚的なフィードバックを即座に提供するためのトースト通知システムを導入しました。

## 目的
- コピー完了をユーザーに分かりやすく伝える。
- 個別のコンポーネント（ChunkCardなど）が持っていた状態フィードバックのロジックを共通化し、UIの一貫性を高める。

## 実装詳細

### 1. ToastProvider と useToast フック
`src/components/Toast.tsx` を作成し、アプリケーション全体でトーストを表示できる仕組みを構築しました。
- `ToastProvider`: アプリケーションのルート（`App.tsx`）でラップし、通知の表示領域と状態を管理します。
- `useToast`: 任意のコンポーネントから `showToast(message)` を呼び出すためのカスタムフックです。

### 2. UI/UX の改善
- **通知のデザイン**: 画面下部中央に、チェックアイコン付きの丸角バッジとして表示されます。
- **ChunkCard の変更**: 
    - 従来はコピーボタン自体が「コピーしました（緑色）」に変化していましたが、トースト通知の導入に伴い、ボタンの状態変化を廃止しました。
    - これにより、ボタンをクリックした瞬間のフィードバックがトーストとして表示され、ボタン自体のレイアウト崩れや色の煩雑さを解消しました。

## 使用方法
```tsx
import { useToast } from './components/Toast';

const MyComponent = () => {
  const { showToast } = useToast();

  const handleClick = () => {
    // 処理...
    showToast("完了しました！");
  };
}
```
